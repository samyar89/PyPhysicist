# پرسش‌های متداول

## آیا این کتابخانه برای استفاده‌ی آموزشی مناسب است؟
بله. هدف اصلی همین است که فرمول‌ها ساده، واضح و سریع قابل استفاده باشند و مثال‌ها هم آموزشی باشند.

## اگر تابع مورد نیازم را پیدا نکردم چه کنم؟
می‌توانید پوشه‌های مربوط به شاخه‌ی فیزیک را بررسی کنید یا مستقیماً در کد جستجو کنید. معمولاً نام تابع نزدیک به نام فرمول است. اگر فرمول وجود ندارد، به‌راحتی می‌توانید یک تابع جدید اضافه کنید.

## باید واحدها را خودم مدیریت کنم؟
اگر ورودی‌های شما فقط عدد هستند، نیازی نیست واحدها را مدیریت کنید. اما اگر از `Quantity` استفاده کنید، خروجی هم واحددار می‌شود و خطاهای واحدی کمتر می‌شود.

## چطور می‌توانم کتابخانه را توسعه بدهم؟
بهترین راه این است که یک تابع جدید با نام واضح و آرگومان‌های نام‌دار اضافه کنید و آن را در `__init__.py` اکسپورت کنید تا از سطح بالا قابل دسترسی باشد.

## آیا پشتیبانی از NumPy دارد؟
بیشتر توابع بر پایه‌ی عملیات ساده هستند و معمولاً با آرایه‌های NumPy هم کار می‌کنند.

## آیا می‌توانم از خروجی LaTeX استفاده کنم؟
بله. با `LatexConverter` می‌توانید معادله‌ها را به صورت LaTeX دریافت کنید و در گزارش یا مقاله استفاده کنید.

## چرا بعضی تابع‌ها با خطا اجرا می‌شوند؟
دلایل رایج:

- پارامتر لازم را نداده‌اید.
- نوع داده یا واحد اشتباه وارد کرده‌اید.
- از تابع‌هایی مثل مقاومت سری/موازی بدون ورودی استفاده کرده‌اید.

## چطور مطمئن شوم نتایج درست هستند؟
- ورودی‌ها را با واحدهای واقعی وارد کنید.
- از ثابت‌های فیزیکی آماده استفاده کنید.
- با یک محاسبه‌ی دستی کوچک، نتیجه را بررسی کنید.

> اگر سؤال دیگری دارید، پیشنهاد می‌شود مستندات API را بررسی کنید یا به ساختار ماژول‌ها سر بزنید.
